Jsp 내장객체

request

  • client 측 자료를 서버로 전달 (톰캣의 내장 객체)
  • request 객체는 클라이언트 측의 데이터를 서버로 전송
  • 클라이언트 컴퓨터 ip, 현재 실행중일 파일의 경로와 파일명, 서버url, port등도 구할 수 있음
  • <a href="url?변수=데이터&변수=데이터..."> </a>
    • ?로 처리하는 것을 쿼리 스트링이라하며,  공백을 사용할 수 없음

세션

  • 서버측에서 관리되는 정보
  • 보안에 취약한 쿠키를 보완하기 위해 사용
  • 브라우저가 종료되면 데이터 삭제
  • 내장 객체이므로 객체를 따로 만들지 않아도됨
  • 아무것도하지 않고 정해진 시간이 지나면 세션이 삭제됨
  • 웹컨테이너에 저장됨
  • 관련 메서드
    • session.invalidate( ) : 세션을 삭제
    • seesion.setArrtibute(”변수명”, “값”) : 세션영영에 정보 기록
    • session.getAttribute(”변수명”) : 세션에 저장되어 있는 변수의 값을 보여줌
    • response.sendRedirect(request.getContextPath()+"/") : 세션 자동 할당
  • web.xml에서 세션 시간을 결정 (분단위)

ex) 로그인유지, 로그아웃 후 세션 삭제

에러발생시 호출하는 페이지

에러발생시 호출

  • <%@ page errorPage=" 페이지 주소" %> 를 지시부에 작성
  • 에러 발생시 “페이지 주소”롤 따라가 해당 페이지로 이동하게됨
  • 이동할 페이지에 <%@ page isErrorPage="true" %>를 지시부에 작성

인크루드(Include)

  • include가 가능한 파일
    • .jsp
      • <jsp:include page=””/>
      • 각기 다른 파일을 가져다 쓰는 거기때문에 데이터의 연동이 이루어지지 않는다.
    • .jspf (조각모음)
      • <%@ include file=”” %>
      • include 당하는 녀석들
      • 데이터의 연동이 이루어진다. BUT!! 실행 순서가 있기때문에 이점을 유의!!
